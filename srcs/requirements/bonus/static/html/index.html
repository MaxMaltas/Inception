🔹 Bloque 2: Dockerfiles y Buenas Prácticas

🛠️ 4. Crear Dockerfiles

  • Sintaxis básica de un Dockerfile.
  • Comandos importantes: FROM, RUN, COPY, WORKDIR, CMD, ENTRYPOINT, EXPOSE, ENV.
  • Buenas prácticas:
    • No usar procesos infinitos.
    • No poner contraseñas dentro del Dockerfile.
    • Entender el rol de PID 1. ### 5. Construcción de imágenes personalizadas
  • docker build -t nombre .
  • Probar imágenes propias con docker run.

🔹 Bloque 3: Docker Compose

🛠️ 6. ¿Qué es Docker Compose?

  • Ventajas de definir múltiples servicios.
  • Sintaxis del archivo docker-compose.yml.
  • Estructura básica: services, build, volumes, networks, env_file, depends_on, restart.

🧪 7. Comandos esenciales de Docker Compose

  • docker-compose up --build
  • docker-compose down
  • docker-compose logs -f
  • docker-compose exec nombre_servicio bash

🔹 Bloque 4: Volúmenes, Redes y Variables

💾 8. Volúmenes Docker

  • Volúmenes persistentes para datos (ej. MariaDB y WordPress).
  • Crear y montar volúmenes desde docker-compose.yml.

🌐 9. Redes Docker

  • Redes bridge personalizadas para conectar servicios.
  • Definición en docker-compose.yml.

🔐 10. Variables de entorno y Docker Secrets

  • Crear archivo .env.
  • Uso de variables con ${VARIABLE} en docker-compose.yml y Dockerfile.
  • Buenas prácticas de seguridad: secrets, .gitignore, evitar hardcoding de contraseñas.
