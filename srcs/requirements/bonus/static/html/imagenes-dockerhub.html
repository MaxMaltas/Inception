🧱 ¿Qué es una imagen Docker?

Una imagen es un archivo de solo lectura que contiene: - Un sistema de archivos (por ejemplo, Alpine o Debian), - Archivos del proyecto o servicio (como WordPress), - Dependencias necesarias (como PHP, NGINX, MariaDB), - Y las instrucciones de configuración (vía Dockerfile).

📌 Es el “molde” del que se crean los contenedores.

📦 ¿Y qué es un contenedor?

Un contenedor es una instancia en ejecución de una imagen.
Podemos decir:

🔁 Imagen = plantilla 📦
🏃‍♂️ Contenedor = ejecución viva de esa plantilla

Cada vez que lanzas un contenedor (docker run nginx), estás creando una copia en ejecución de la imagen nginx.

🧰 ¿Qué es DockerHub?

DockerHub es como GitHub pero para imágenes Docker.
Es un repositorio público donde: - Puedes buscar imágenes oficiales (como nginx, mariadb, wordpress). - Puedes subir tus propias imágenes si tienes una cuenta. - Se utiliza por defecto cuando haces un docker pull <nombre>.

🔍 Cómo buscar imágenes:

docker search nginx
  • Esto muestra una lista de imágenes disponibles en DockerHub relacionadas con “nginx”.

📥 Cómo descargar una imagen:

docker pull nginx
  • Esto descarga la imagen oficial más reciente de nginx.

🧪 Cómo ejecutar esa imagen:

docker run -d --name test-nginx -p 8080:80 nginx
  • Esto lanza un contenedor a partir de la imagen descargada.

⚠️ Importante para el proyecto Inception:

❌ ¡No puedes usar directamente imágenes preconstruidas en el proyecto Inception!
✅ Tienes que construirlas tú mismo con Dockerfile, aunque sí puedes usar como base alpine o debian:

FROM alpine:latest

Entonces: - Puedes usar Alpine o Debian como base (FROM alpine o FROM debian). - Luego tú debes instalar y configurar WordPress, MariaDB, NGINX, etc.
